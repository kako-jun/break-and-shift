# ロードマップ

## 実装予定のコンテンツ

---

### PixiJS シミュレーションの拡張

3D サイコロを含む各シミュレーター（チンチロリン / クレーンゲーム / じゃんけん / 雨の中の歩行 / 生存者バイアス / 先祖の確率）は、React コンポーネントとして直接 mount する方針（PixiJS、chinchirorin は Three.js + cannon-es）。本実装は別 Issue で順次。

今後の拡張として以下を検討している。

- cannon-es の物理エンジンによるサイコロのより厳密なシミュレーション
- 複数カメラアングル切り替えなど experiment の UI 強化
- 「サイコロは本当にランダムか」をさらに精密に検証できる素材へ

---

### 宝くじシミュレーション（拡張版）

第二章の宝くじシミュレーターは、現在は基本的な購入・収支計算のみ。

拡張として以下を検討している。

- モンテカルロシミュレーション：数万回の試行を一括実行し、収支分布をヒストグラムで表示
- 「一生分」シミュレーション：年に何枚買い続けると、何年後に1等に当たる確率が何%になるかを計算
- 損益曲線の長期グラフ：運良く当たった場合でも長期的には損をする理由を視覚化

---

### UFOキャッチャーAI

クレーンゲームの現在の実装は、失敗回数に応じて握力が線形上昇するモデル。

計画として、強化学習エージェントがクレーンゲームを学習する様子を可視化する実装を検討している。

- エージェントがどの戦略を試して、どこで失敗して、最適解に収束していくかを可視化
- 「人間の感覚的な戦略」と「確率的に最適な戦略」の差を示す素材になる
- PixiJS 側の物理処理と統合し、握力・摩擦・物体の重心を現実的にモデリング

---

### ゲームフロー分析（流れがあるか調べるゲーム）

「麻雀やスロットには流れがある」という信仰を検証するためのコンテンツ。

第五章の麻雀牌シミュレーションを発展させ、以下を実装する。

- 試行データに対して自己回帰分析または連の検定を行い、「流れ」と呼ばれるパターンが統計的に存在するかどうかを判定
- 「流れに乗っているとき」に見える偏りが、実際にはランダムな揺らぎであることを統計的に示す
- ユーザーが「流れあり」「なし」を目で判断するゲームと、統計の答えを比べるインタラクション

---

### じゃんけんトーナメント

第三章の進化シミュレーターの発展系として、じゃんけんのトーナメント表を自動生成するコンテンツ。

- N人が参加するトーナメントを生成し、結果をHTMLで表示
- プレイヤー自身の手も設定でき、どこまで勝ち進めるかを体験できる
- 「優勝者は最強の手を出したのか、それとも運だったのか」を確率で検証
